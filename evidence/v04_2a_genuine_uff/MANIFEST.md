# 2a (HKUST-1 + CO2) genuine-UFF evidence (v0.4.2 rebuild, 2026-06-21)

Promoted from validated production-count scratch runs (TASK 11/14, RASPA3) to committed evidence
(TASK 15), so the v0.4.2 numbers trace to committed JSON. Parallel to the 6c
`evidence/v04_6c_olson_clean/` promotion.

## Why this rebuild
The originally-locked 2a recipe was generated by a RASPA3 input generator that hard-coded a
zeolite/cation LJ table and unconditionally stamped `source="UFF"` on the MOF framework atoms,
even though the light-atom values were not genuine UFF (framework C carried the Harris-Yung
CO2-carbon eps 28.1 vs UFF 52.8, O carried TraPPE-zeo 53.0 vs UFF 30.2, H a wrong eps; only Cu was
genuine UFF). This produced the false "UFF Cu under-binds" reading (as-locked dlogKH = -0.759). The
generator was fixed (`input_writer._uff_lj_K`, fail-loud; zeolite path byte-identical).

## Recipe (corrected)
- Framework: **genuine plain Rappe UFF (1992)** per element (Cu 2.516/3.114, C 52.84/3.431,
  O 30.19/3.118, H 22.14/2.571) + Nazarian 2016 DDEC charges from the CIF.
- Guest: EPM2/Harris-Yung CO2 as locked; Ewald; shifted-truncated; 12 A cutoff.
- Structure: `FIQCEN_clean_min_charges.cif` (HKUST-1, primitive rhombohedral cell, 156 atoms).

## Result (vs reference K_H 7.0, Q_st 30.0)
- Engine: **RASPA3 v3.0.29** (binary sha256 `2b4132becc0f38de...`), **12 seeds x 400,000 cycles**, T = 298.15 and 323.15 K.
- K_H(298) is reported as a **range** because of an open-Cu deep-well sampling tail:
  median 8.33 / trimmed geomean 9.53 / geomean 10.80 mol/kg/bar (per-seed 6.6-11.4, with 3 of 12 at
  24-28 from rare deep-well insertions). dlogKH spans +0.075 (median) to +0.188 (geomean).
- Q_st (two-point van't Hoff, 298/323) = 33.9 kJ/mol; dQst = +3.9.
- Verdict is robust to the K_H estimator: **Tier-B PASS** (dlogKH <= 0.4 by all estimators;
  dQst +3.9 keeps it out of Tier-A regardless).
- Full per-seed values + input sha256 in `results.json`; the run is reproducible from the committed inputs (force_field.json, simulation.json, CIF; sha256 recorded in results.json). For brevity the multi-MB full RASPA3 output logs are not vendored here; the per-seed Henry coefficients are in `results.json`.

## Caveat (RASPA3-only)
The native charged-Ewald estimator shows a large empirical discrepancy on these oblique MOF
primitive cells (~4x on this HKUST-1 cell; root cause not established -- see the manuscript
limitation), so it does **not** corroborate these numbers. The verdict rests on RASPA3 alone.

Superseded as-locked: dlogKH = -0.759 / dQst = -9.07 (retained in the case matrix as the documented
buggy case).
